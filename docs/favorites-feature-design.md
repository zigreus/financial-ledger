# 거래내역 즐겨찾기 기능 설계

## 1. 개요

자주 입력하는 거래 패턴을 즐겨찾기로 저장해두고, 입력 폼에서 원클릭으로 불러와 빠르게 등록하는 기능.

**핵심 흐름:**  
폼에서 별표 클릭 → 즐겨찾기 자동 등록 → 이후 폼 상단 칩에서 선택 → 필드 자동 채움 → 필요시 수정 → 저장

---

## 2. 데이터 모델

### 신규 테이블: `favorite_transactions`

```sql
CREATE TABLE IF NOT EXISTS favorite_transactions (
  id              INTEGER PRIMARY KEY AUTOINCREMENT,
  name            TEXT NOT NULL,          -- 즐겨찾기 이름 (자동 또는 수동 입력)
  payment_method  TEXT NOT NULL,          -- 결제수단
  budget_category TEXT NOT NULL,          -- 카테고리
  sub_category    TEXT DEFAULT '',        -- 세부카테고리
  detail          TEXT DEFAULT '',        -- 세부내역
  amount          INTEGER NOT NULL,       -- 금액 (정수, 수식 계산 후 저장)
  sort_order      INTEGER DEFAULT 0,      -- 정렬 순서 (드래그 재정렬)
  use_count       INTEGER DEFAULT 0,      -- 사용 횟수 (통계)
  last_used_at    TEXT,                   -- 마지막 사용일
  created_at      TEXT DEFAULT (datetime('now','localtime'))
);
```

**기존 transactions 테이블과의 관계:** 1:N (즐겨찾기 1개 → 여러 번 실제 거래로 등록됨)  
**저장 위치:** 기존 SQLite DB (ledger.db) 내 신규 테이블 — OneDrive 자동 동기화 그대로 적용.  
**`discount_amount` 미저장 이유:** 할인은 폼 불러올 때 결제수단 기준으로 자동 재계산.

## 3. UI/UX 설계

### 3-1. 거래 추가 폼 — 별표 즐겨찾기 등록

#### 별표 위치: 모달 헤더 제목 옆

현재 헤더 구조: `[거래 추가]` + `[✕]`  
변경 후: `[거래 추가 ☆]` + `[✕]`

```
┌─────────────────────────────────────┐
│  거래 추가 ☆                    ✕  │   ← 별표를 제목 바로 옆에
│  ─────────────────────────────────  │
│  날짜: 2026-04-26                   │
│  결제수단: [우리카드    ▼]          │
│  카테고리: [교통비  ▼][대중교통 ▼]  │
│  세부내역: 출퇴근                    │
│  금액: 3,550                        │
│  ─────────────────────────────────  │
│                     [취소]  [추가]  │
└─────────────────────────────────────┘
```

제목 옆 위치를 선택한 이유:
- "이 거래를 즐겨찾기"라는 맥락이 명확하게 전달됨
- 필드 입력 영역과 분리되어 있어 혼동 없음
- ✕ 닫기 버튼과는 충분한 간격 유지

#### 별표 동작 (크롬 북마크 방식)

| 상태 | 아이콘 | 동작 |
|------|--------|------|
| 미등록 | ☆ (빈 별, 회색) | 클릭 시 즉시 ★ 등록 + 버블 팝업 열림 |
| 등록됨 | ★ (채워진 별, 강조색) | 클릭 시 버블 팝업 열림 |

#### 버블 팝업

☆ 클릭(신규 등록)과 ★ 클릭(수정/해제) 모두 같은 팝업 사용.

```
거래 추가 ★                              ✕
          ┌──────────────────────┐
          │ [출퇴근            ] │  ← 이름 바로 편집 가능
          │ [해제]      [완료]  │
          └──────────────────────┘
```

- 팝업 외부 클릭 → 팝업 닫힘, 이름은 마지막 입력값으로 저장
- **[완료]**: 이름 저장 후 팝업 닫힘
- **[해제]**: 즐겨찾기 삭제, ★ → ☆, 팝업 닫힘 (별도 확인 배너 없음 — 팝업에서 의도를 명확히 했으므로)

**이름 자동 생성 규칙 (☆ 클릭 시 초기값):**
- 세부내역이 있으면 → 세부내역 그대로 (`"출퇴근"`)
- 세부내역이 없으면 → `"결제수단-세부카테고리"` 형태 (`"우리카드-대중교통"`)

**즐겨찾기 등록 최소 조건:**

결제수단 · 카테고리 · 세부카테고리 · 금액 4개 중 **2개 이상** 입력되어 있어야 등록 가능.  
목적이 입력 속도 단축이므로, 2개만 채워도 그 2개는 자동으로 불러와지므로 충분히 유용함.

| 입력된 필드 수 | ☆ 상태 |
|---------------|--------|
| 0 ~ 1개 | 비활성(disabled), 흐린 색상 |
| 2개 이상 | 활성, 클릭 가능 |

**조건 미충족 시 UX:**
- ☆ 비활성화(disabled) + 흐린 색상으로 표시
- hover/탭 시 툴팁: "결제수단, 카테고리, 세부카테고리, 금액 중 2개 이상 입력하면 즐겨찾기를 등록할 수 있어요"
- 조건 충족 순간 ☆ 자동 활성화

**수정 폼에서 별표 동작:**
- 편집 대상 거래가 즐겨찾기로 등록된 적 없으면 → ☆
- 편집 대상 거래 내용 기반으로 매칭되는 즐겨찾기가 있으면 → ★

### 3-2. 입력 폼 상단 — 즐겨찾기 칩으로 불러오기

즐겨찾기가 1개 이상 존재할 때 폼 상단에 가로 스크롤 칩 표시.

```
┌─────────────────────────────────────┐
│  ★ [출퇴근] [고속도로(와이프)] [편의점] →  │
└─────────────────────────────────────┘
│  날짜: 2026-04-26                   │
│  ...                                │
```

- 즐겨찾기 없으면 영역 미표시 (폼 복잡도 유지)
- 칩 클릭 시 폼 전체 필드 채움 (날짜·일정은 유지)
- 선택된 칩 하이라이트, 필드 수정해도 하이라이트 유지

**불러올 때 채워지는 필드:**

| 필드 | 동작 |
|------|------|
| 날짜 | 유지 (오늘 날짜) |
| 결제수단 | 즐겨찾기 값으로 채움 |
| 카테고리/세부 | 즐겨찾기 값으로 채움 |
| 세부내역 | 즐겨찾기 값으로 채움 |
| 금액 | 즐겨찾기 값으로 채움 |
| 할인 | 결제수단 기준 자동 재계산 |
| 일정 | 유지 (현재 선택 일정 그대로) |

현재:
```

```

변경 후:
```
[지출계획] → 서브탭: [즐겨찾기] [정기지출] [예산]
```

- 지출계획 탭 진입 시 기본 서브탭을 **즐겨찾기**로 변경
- `financeSubTab` 기본값: `'budget'` → `'favorites'`로 변경

#### 즐겨찾기 관리 화면

```
┌──────────────────────────────────────┐
│  [즐겨찾기] [정기지출] [예산]         │
│  ──────────────────────────────────  │
│  ≡  출퇴근                           │
│     우리카드 · 교통비/대중교통        │
│     3,550원 · 12회 사용    ✏  🗑     │
│                                      │
│  ≡  고속도로 (와이프)                 │
│     우리카드 · 교통비/대중교통        │
│     1,900원 · 5회 사용     ✏  🗑     │
│                                      │

```

- `≡` 드래그로 순서 변경 → 폼 상단 칩 순서에 반영

- 🗑 클릭 → "즐겨찾기를 삭제하시겠습니까?" 확인 후 삭제
- "새 즐겨찾기 추가" → 폼 펼침 (정기지출 추가 패턴과 동일)
- 사용 횟수 표시: "N회 사용"

### 3-4. 스마트 패턴 자동 추천

즐겨찾기를 직접 등록하지 않아도, 과거 거래 패턴을 분석해 입력을 도와주는 기능.

#### A. 결제수단 자동 선택 (항상 동작)

세부카테고리 선택 시, 해당 조합의 과거 거래를 분석해 결제수단을 자동으로 채움.

**발동 조건 (모두 충족 시):**
- 해당 세부카테고리의 최근 거래가 **10건 이상**
- 그 중 동일 결제수단 비율이 **80% 이상**
- 해당 결제수단이 **히든(is_hidden=1) 처리되지 않은** 경우

**히든 결제수단 처리:**
- 설정 > 결제수단에서 히든 처리한 결제수단은 자동 선택 후보에서 제외
- 히든 결제수단이 패턴 1위여도 무시하고, 비히든 중 다음 순위 결제수단으로 평가

**동작:**
- 조건 충족 → 결제수단 자동 선택 (사용자가 바꿀 수 있음)
- 조건 미충족 → 기존과 동일 (자동 선택 없음)

즐겨찾기와 독립적으로 항상 동작. 즐겨찾기가 없어도 효과 있음.

**설정에서 감지된 패턴 확인:**

설정 > 지출계획 > 즐겨찾기 탭 하단에 "자동 패턴" 섹션 추가.

```
┌──────────────────────────────────────┐
│  [즐겨찾기] [정기지출] [예산]         │
│  ── 즐겨찾기 목록 ──────────────────  │
│  ...                                 │
│                                      │
│  ── 자동 결제수단 패턴 ─────────────  │
│  대중교통 → 우리카드 (89%, 18건)  ●─  🗑 │
│  고속도로 → 우리카드 (100%, 5건)  ●─  🗑 │
│  편의점   → 현금    (90%, 11건)   ─○  🗑 │
└──────────────────────────────────────┘
```

| 액션 | 동작 |
|------|------|
| **토글 켜짐 (●─)** | 세부카테고리 선택 시 해당 결제수단 자동 선택 동작 |
| **토글 꺼짐 (─○)** | 자동 선택 안 함 (패턴 이력은 유지) |
| **🗑** | 이력 초기화. 이후 패턴이 다시 쌓이면 재감지됨 |

- 감지된 패턴이 없으면 해당 섹션 미표시
- 섹션 하나로 활성/비활성 통합 표시

#### B. 즐겨찾기 자동 추천 알림 (패턴 감지)

동일한 패턴이 충분히 쌓이면 즐겨찾기 등록을 제안.

**발동 조건:**
- (결제수단 + 카테고리 + 세부카테고리 + 금액) 조합이 **최근 10건 중 8건 이상** 일치
- 해당 조합이 아직 즐겨찾기에 등록되지 않은 경우

**추천 알림 UI:**
```
자주 쓰는 패턴이에요. 즐겨찾기에 추가할까요?
우리카드 · 교통비/대중교통 · 3,550원
[다시 보지 않기]   [나중에]   [추가]
```

- **[추가]**: 이름 자동 생성 후 즉시 등록
- **[나중에]**: 해당 패턴으로 **10건 더 쌓인 후** 재추천
- **[다시 보지 않기]**: 해당 패턴에 대해 영구 중단

**추천 횟수 제한:**

| 거절 횟수 | 다음 추천 시점 |
|-----------|---------------|
| 1회 거절 | 해당 패턴 10건 후 |
| 2회 거절 | 해당 패턴 10건 후 |
| 3회 거절 | **영구 중단** (더 이상 추천 안 함) |

3번 거절하면 [다시 보지 않기]를 누르지 않아도 자동으로 영구 중단.  
[다시 보지 않기]는 언제든 즉시 영구 중단하는 명시적 수단.

> 두 방식의 역할이 다름: 횟수 제한은 "나도 모르게 귀찮음이 해소", [다시 보지 않기]는 "지금 당장 강하게 거부"

**금액 일치 기준:**
- 완전히 같은 금액만 카운트 (±0원)
- 금액이 매번 다르면 패턴 미감지 (결제수단 자동선택 A만 동작)

**추천 상태 저장:** 별도 테이블 `pattern_suggestion_log`로 관리 (구현 범위 참고)

## 4. 구현 범위

### 신규 테이블

**`favorite_transactions`** — 즐겨찾기 저장 (2. 데이터 모델 참고)

**`pattern_suggestion_log`** — 자동 추천 알림 상태 추적

```sql
CREATE TABLE IF NOT EXISTS pattern_suggestion_log (
  pattern_key        TEXT PRIMARY KEY,  -- "결제수단|세부카테고리|금액" 조합 키
  suggest_count      INTEGER DEFAULT 0, -- 총 추천 횟수
  dismiss_count      INTEGER DEFAULT 0, -- 거절 횟수 (나중에 클릭)
  never_suggest      INTEGER DEFAULT 0, -- 1이면 영구 중단
  last_suggest_tx_count INTEGER DEFAULT 0 -- 마지막 추천 시점의 해당 패턴 거래 수
);
```

### DB 함수 (`dbManager.js`)

**즐겨찾기 CRUD:**
- `initFavoriteTransactions(db)` — 테이블 생성, 기존 DB 마이그레이션 안전 처리
- `getFavorites(db)` — sort_order 순 전체 조회
- `addFavorite(db, name, txTemplate)` — 신규 등록
- `updateFavorite(db, id, fields)` — 수정
- `deleteFavorite(db, id)` — 삭제
- `recordFavoriteUse(db, id)` — use_count +1, last_used_at 갱신
- `reorderFavorites(db, id, newPosition)` — 정렬 순서 변경

**스마트 패턴:**
- `getAutoPaymentMethod(db, subCategory)` — 세부카테고리 기준 자동 선택 결제수단 반환 (히든·토글 꺼짐 제외)
- `getDetectedPatterns(db)` — 설정 화면용 감지된 패턴 목록 반환 (토글 상태 포함)
- `togglePattern(db, patternKey, enabled)` — 패턴 토글 켜짐/꺼짐 전환
- `deletePattern(db, patternKey)` — 감지 이력 초기화 (행 삭제, 이후 재감지 가능)
- `getSuggestionLog(db, patternKey)` — 즐겨찾기 추천 알림 상태 조회
- `recordSuggestDismiss(db, patternKey)` — 거절 기록 (dismiss_count +1, 3회 시 never_suggest=1)
- `recordNeverSuggest(db, patternKey)` — 영구 중단 설정 ([다시 보지 않기])

### 신규 컴포넌트

- `FavoriteChips.js` / `FavoriteChips.css` — 폼 상단 칩 선택 영역
- `FavoritesManager.js` / `FavoritesManager.css` — 설정 내 관리 화면 (정기지출 패턴 참고)

### 기존 컴포넌트 수정

| 파일 | 변경 내용 |
|------|-----------|
| `TransactionForm.js` | FavoriteChips 삽입, 별표 체크박스 + 이름 입력창 추가 |
| `SettingsView.js` | 서브탭에 즐겨찾기 추가, 탭 순서 변경, 기본 서브탭 변경 |
| `App.js` | favorites 상태 관리, CRUD 핸들러 추가 |

## 5. 고려사항

| 항목 | 결정 |
|------|------|
| 즐겨찾기 최대 개수 | 제한 없음 (칩 가로 스크롤로 처리) |
| 결제수단/카테고리 삭제 시 | 즐겨찾기 값 유지, 폼 불러올 때 "삭제된 항목" 경고 표시 |
| 일정(event_id) 저장 여부 | 저장하지 않음 — 일정은 상황마다 달라지므로 폼에서 선택 |
| 금액 수식 저장 여부 | 계산된 정수값만 저장 (수식 자체는 미저장) |
| 가족 공유 | 하나의 DB이므로 자동 공유됨 (구분 불필요) |
| 마이그레이션 | `CREATE TABLE IF NOT EXISTS`로 기존 DB에 무해하게 추가 |
| 이름 중복 | 허용 (같은 이름 여러 개 가능, 칩에서는 이름 그대로 표시) |

## 6. 구현 우선순위

1. **DB 테이블 + CRUD 함수** — 기반
2. **TransactionForm 별표 즐겨찾기 등록** — 핵심 등록 경로
3. **폼 상단 즐겨찾기 칩** — 핵심 사용 경험
4. **설정 > 지출계획 탭 구조 변경 + 즐겨찾기 관리 화면** — 수정/삭제/재정렬
