# AI 자동입력 기능 기획서

## 개요

거래내역·일정 입력 피로도를 낮추기 위한 AI 보조 기능 묶음.
자연어 입력, 영수증 OCR, 과거 패턴 기반 자동완성, 정기지출 자동 감지를 포함한다.

---

## 배경 및 목적

거래내역·일정 추가 시 결제수단, 카테고리, 날짜, 금액 등 여러 필드를 일일이 선택·입력해야 한다. 정기지출 자동등록 기능이 있지만 일회성·비정형 지출은 여전히 수동 입력이 필요하다.

**목표:** 입력 방식을 다양화하여 데이터 추가 마찰을 최소화한다.

---

## 기능 전체 목록 및 AI 필요 여부

| 기능 | AI 필요 여부 | 추천 구현 방식 |
|------|------------|--------------|
| 거래내역 자연어 입력 | 부분적 | 규칙 기반 (날짜·금액·결제수단) + 경량 API (카테고리 추론) |
| 일정 자연어 입력 | 부분적 | 규칙 기반으로 대부분 처리 가능 |
| 영수증 이미지 → 거래내역 | 필요 | Vision LLM API (Gemini Vision) |
| 과거 패턴 자동완성 | 불필요 | SQL 이력 조회 + 문자열 매칭 |
| 정기지출 패턴 자동 감지 | 불필요 | SQL 통계 분석 |

## SLM(소형 언어 모델) 브라우저 실행 검토

자연어 파싱을 외부 API 없이 처리하기 위해 브라우저 내 SLM 실행을 검토했다.

### 기술 옵션

| 방식 | 대표 라이브러리 | 모델 크기 | WebGPU 필요 | 한국어 지원 |
|------|--------------|---------|------------|-----------|
| WebLLM / MLC-LLM | `@mlc-ai/web-llm` | 500MB~4GB | 필수 | Phi-3.5, Qwen2.5 일부 지원 |
| Transformers.js | `@xenova/transformers` | 100~500MB | 불필요 (WebAssembly) | 제한적 (NER, 분류 모델) |
| 규칙 기반 파싱 | 자체 구현 | 0MB | 불필요 | 완전 지원 |

### 결론: 브라우저 SLM은 이 앱에 부적합

- **모바일 우선 앱**인데 WebGPU는 모바일 브라우저 지원이 불안정하고 성능 편차가 큼
- 첫 실행 시 수백 MB 모델 다운로드 → 사용성 저하
- 소형 모델의 한국어 구조화 추출 정확도는 검증되지 않음
- 이 앱의 입력 패턴(날짜·금액·결제수단)은 규칙 기반으로 80~90% 커버 가능

**현실적 대안:** 규칙 기반 파싱으로 명시적인 필드를 처리하고, 카테고리 추론처럼 맥락 판단이 필요한 부분만 **Groq 무료 API** (Llama 3.2 기반) 호출로 보완한다. 사실상 비용 없이 LLM 수준의 결과를 얻을 수 있다.

## 기능 상세

### 1. 거래내역 자연어 입력

**입력 예시:**
```
어제 이마트 장보기 국민카드 43500원
스타벅스 아메리카노 삼성카드 4500원
지난주 금요일 주유 현대카드 65000원
```

**파싱 전략:**

| 필드 | 처리 방식 | 비고 |
|------|---------|------|
| 날짜 | 규칙 기반 | "어제", "지난 금요일", "n월 n일" 등 패턴 매칭 |
| 금액 | 규칙 기반 | "43500원", "4만5천원", "4.5만" 등 한국어 숫자 표현 |
| 결제수단 | 규칙 기반 | DB 등록 목록 대상 퍼지 매칭 |
| 카테고리 | 경량 API | 상점명·키워드 → 카테고리 추론 (규칙으로 어려운 케이스) |
| 상세내용 | 규칙 기반 | 상점명, 품목명 추출 |
| 할인금액 | 규칙 기반 | "2000원 할인", "10% 할인" 패턴 |

**미추출 필드:** 빈값으로 두고 사용자 직접 입력.

---

### 2. 일정 자연어 입력

**입력 예시:**
```
5월 제주도 여행 5/3~5/6
다음달 15일 부모님 결혼기념일
6월 오사카 출장 6/10~6/14 일본 엔화
```

**파싱 전략:** 대부분 규칙 기반으로 처리 가능.

| 필드 | 처리 방식 | 비고 |
|------|---------|------|
| 제목 | 규칙 기반 | 날짜·키워드 제거 후 나머지 |
| 일정 유형 | 규칙 기반 | "여행", "출장" → trip / "결혼기념일", "생일" → occasion |
| 시작일 / 종료일 | 규칙 기반 | "5/3~5/6", "다음달 15일" 등 |
| 국가 / 통화 | 규칙 기반 | "일본" → JPY, "미국" → USD 등 매핑 테이블 |
| 메모 | 잔여 텍스트 | 위 필드 추출 후 남은 내용 |

### 3. 영수증 이미지 → 거래내역 추출

**흐름:**
```
[영수증 촬영 / 이미지 업로드]
    ↓
Vision LLM API (이미지 + 추출 지시 프롬프트)
    ↓
거래내역 JSON 반환
    ↓
TransactionForm 자동완성 → 사용자 확인 후 저장
```

**구현 방식:**

Gemini Vision API (`gemini-2.0-flash`) 사용. 이미지와 텍스트를 함께 처리하는 멀티모달 모델로, 영수증의 가게명·품목·금액·날짜를 구조화 추출한다.

```javascript
// 영수증 이미지 → 거래내역 JSON
parseReceiptImage(imageBase64, { paymentMethods, categories, today })
  → { date, payment_method, budget_category, sub_category, detail, amount, discount_amount }
```

**비용:** Gemini 2.0 Flash 이미지 처리 기준 건당 약 $0.001~0.003. 월 50장 기준 $0.05~0.15 수준.

**Tesseract.js(브라우저 OCR) 사용 불가 이유:** 한국어 OCR 정확도가 낮고 영수증 레이아웃 구조 이해가 불가능하여 필드 추출 결과가 불안정하다.

### 4. 과거 패턴 자동완성

**AI 불필요. SQL 이력 조회만으로 구현.**

거래 입력 폼에서 상세내용(가게명 등)을 타이핑하면, 동일 가게의 최근 거래 이력에서 결제수단·카테고리를 자동 제안한다.

**동작 예시:**
```
사용자 입력: "스타벅스"
  ↓
DB에서 detail LIKE '%스타벅스%' 최근 거래 조회
  ↓
가장 자주 쓴 결제수단: 삼성카드
가장 자주 쓴 카테고리: 식비 > 카페
  ↓
해당 필드 자동 제안 (사용자가 수락/무시 선택)
```

**구현 포인트:**
- 디바운스 처리로 타이핑 중 과도한 조회 방지
- 이력이 없는 경우 제안 없이 일반 입력 유지
- 제안 수락 시 해당 패턴 가중치 증가 (조회 횟수 기반)

---

### 5. 정기지출 패턴 자동 감지 및 등록 제안

**AI 불필요. SQL 통계 분석으로 구현.**

거래 이력에서 동일한 금액·가게가 일정 간격으로 반복되는 패턴을 감지하고, 정기지출 등록을 제안한다.

**감지 알고리즘:**
```
transactions 테이블에서 그룹핑:
  GROUP BY detail, amount, payment_method
  → 3회 이상 반복된 항목 추출

날짜 간격 분석:
  간격이 28~31일 → 월 정기지출 후보
  간격이 6~8일  → 주 정기지출 후보
  간격이 360~366일 → 연 정기지출 후보

기존 정기지출(recurring_transactions)과 중복 여부 확인
  → 미등록 패턴만 제안 목록에 포함
```

**UX:**
- 설정 화면 또는 별도 배너에서 "n건의 정기지출 패턴이 발견되었습니다" 알림
- 각 패턴별로 [정기지출로 등록] / [무시] 선택
- 무시한 패턴은 재표시하지 않음 (dismissed 플래그 관리)

## 기술 구현

### 파싱 레이어 구조

```
사용자 입력 (자연어 텍스트)
    ↓
[1단계] 규칙 기반 파서 (자체 구현)
  - 날짜 파싱: 한국어 날짜 표현 → YYYY-MM-DD
  - 금액 파싱: 한국어 숫자 표현 → 정수
  - 결제수단: DB 목록 퍼지 매칭
  - 일정 유형: 키워드 매핑
    ↓
[2단계] 미해결 필드만 API 호출 (필요 시)
  - 카테고리 추론 등 맥락 판단 필요한 경우
  - Groq API (무료) 또는 Gemini API
    ↓
JSON 결과 → 폼 자동완성
```

### 신규 파일

- `src/services/aiService.js` — AI API 호출 및 파싱 로직 전체
  - `parseTransactionText(text, context)` — 거래내역 자연어 파싱
  - `parseEventText(text)` — 일정 자연어 파싱
  - `parseReceiptImage(imageBase64, context)` — 영수증 이미지 파싱
  - `getApiKey()` / `setApiKey(key)` — API 키 관리
- `src/services/patternAnalyzer.js` — 순수 SQL 기반 패턴 분석
  - `getSuggestionsForDetail(db, detail)` — 과거 패턴 자동완성
  - `detectRecurringPatterns(db)` — 정기지출 패턴 감지

### 수정 파일

- `src/components/TransactionForm/TransactionForm.js` — 자연어 입력 UI, 영수증 버튼, 자동완성 제안 추가
- `src/components/CalendarView/CalendarView.js` — EventForm 자연어 입력 UI 추가
- `src/components/SettingsView/SettingsView.js` — AI 제공사 선택 + API 키 설정, 패턴 감지 결과 표시

### 프롬프트 설계 (거래내역)

규칙 기반으로 추출 실패한 필드만 포함해서 호출하므로 토큰 효율이 높다.

```
시스템: 가계부 입력 도우미. 아래 정보를 JSON으로만 반환.
오늘: {today}
카테고리 목록: {categories}

사용자 입력: {text}
미확인 필드: {missingFields}

반환: { "budget_category": "...", "sub_category": "..." }
```

## AI 제공사 비교

> 예상 사용량: 월 300회, 회당 입력 200 + 출력 100 tokens = 월 90,000 tokens

### 공통 주의사항

**Claude.ai Pro, ChatGPT Plus, Gemini Advanced 등 소비자 구독은 API 접근과 완전히 별개다.**
앱에 AI를 붙이려면 API 키를 별도 발급받고 사용량만큼 추가 비용이 발생한다.

### 자연어 파싱용 (텍스트)

| 제공사 | 모델 | 월 예상비용 | 무료 티어 | 브라우저 SDK | 비고 |
|--------|------|-----------|---------|------------|------|
| **Groq** | Llama 3.2 등 | $0 | 500K tokens/일 | OpenAI 호환 | 무료로 충분, 속도 빠름 |
| **Mistral** | Mistral Nemo | ~$0.004 | 1B tokens/월 | 지원 | 무료 티어 압도적 |
| **Google Gemini** | gemini-2.0-flash | ~$0.024 | 있음\* | 지원 (@google/genai) | 영수증 OCR도 함께 처리 가능 |
| **Anthropic** | claude-haiku-4-5 | ~$0.33 | 없음 | 공식 미지원 | 한국어 품질 우수, 브라우저 비공식 |
| **OpenAI** | gpt-4o-mini | ~$0.027 | 없음 | 위험 플래그 필요 | 메리트 적음 |

\* Gemini 무료 티어는 입력 데이터가 Google 제품 개선에 활용될 수 있음.

| 기능 | 권장 |

| API 오류 (네트워크, 인증 등) | 토스트 메시지, 규칙 기반 결과로 폴백 |
| 파싱 실패 (JSON 형식 오류) | 빈 폼으로 폴백, 오류 메시지 표시 |
| 불명확한 입력 | 추출 가능한 필드만 채우고 나머지는 빈값 |
| 영수증 이미지 품질 불량 | "이미지를 더 선명하게 촬영해주세요" 안내 |
| 패턴 이력 부족 (<3건) | 자동완성 제안 미표시 |

---

## 보안 고려사항

- API 키는 localStorage에만 저장. 코드에 하드코딩 금지.
- 입력 텍스트·이미지는 선택한 AI 제공사 API로만 전송되며 별도 서버 저장 없음.
- API 키 입력 필드는 password 타입으로 마스킹.
- 설정 화면에서 API 키 삭제 기능 제공.
- Gemini 무료 티어 사용 시 입력 데이터가 Google 모델 학습에 활용될 수 있음 → 유료 전환 또는 Groq 사용 권장.
- 영수증 이미지에는 민감 정보(카드번호 일부 등)가 포함될 수 있음 → Vision API 전송 전 사용자에게 안내.

## 구현 우선순위

| 단계 | 기능 | 의존성 | 비고 |
|------|------|--------|------|
| 1 | 과거 패턴 자동완성 | 없음 | API 키 불필요, 즉시 구현 가능 |
| 2 | 정기지출 패턴 감지 | 없음 | API 키 불필요, 기존 DB 데이터 활용 |
| 3 | 거래내역 자연어 입력 (규칙 기반) | 없음 | 핵심 기능, API 없이 80% 커버 |
| 4 | 일정 자연어 입력 (규칙 기반) | 없음 | 3단계와 병행 가능 |
| 5 | API 키 설정 UI + aiService.js 기반 구조 | — | API 연동 전 선행 필요 |
| 6 | 자연어 파싱 카테고리 추론 (Groq API) | 5단계 | 규칙 기반 보완 |
| 7 | 영수증 이미지 OCR (Gemini Vision) | 5단계 | 별도 UI 필요 |

## 미포함 범위

- 음성 입력 → 자연어 변환 (Web Speech API 또는 Whisper API 연동)
- 자동완성 제안 수락 이력 기반 개인화 가중치 학습
- 복수 거래 일괄 파싱 (한 번에 여러 줄 입력)
- 영수증 내 복수 품목 → 개별 거래내역 분할 등록
