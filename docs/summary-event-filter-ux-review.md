# 요약 > 일정별 유형 필터 UX 개선 검토

## 현황 및 문제

- **현재 구조:** 유형 필터 버튼 행(전체·일상·여행·경조…) + 일정 드롭다운 2단계 구성
- **문제점:** 유형이 늘어날수록 버튼 행이 좁아져 레이아웃 깨짐, 두 컨트롤의 역할 분리가 직관적이지 않음

---

## A안 — optgroup 통합 드롭다운 ✅ 추천

두 컨트롤(버튼 행 + 이벤트 드롭다운)을 단일 `<select>`로 통합.

```
전체 일정
── [🟡 여행] ────────────
    여행 전체 (합계)
    제주도 여행 (04-10)
    일본 여행 (03-20)
── [🔴 경조] ────────────
    경조 전체 (합계)
    결혼식 (05-01)
── [⚫ 일상] ────────────
    일상 전체 (합계)
    ...
```

- 유형 구분은 `<optgroup>` 헤더로 표현
- 각 그룹 첫 번째 옵션 "○○ 전체" → 기존 유형 필터 역할 대체
- 버튼 행 완전 제거

| 항목 | 평가 |
|------|------|
| 확장성 | ✅ 유형 수 무관하게 레이아웃 고정 |
| 기능 보존 | ✅ 유형별 합계 + 개별 일정 모두 지원 |
| 구현 복잡도 | 낮음 (select value 규칙만 정의) |
| 단점 | optgroup은 스타일링 제한 (색상 도트 표현 불가) |

---

## B안 — 유형 버튼 → 드롭다운 교체

버튼 행만 `<select>`로 교체, 기존 이벤트 드롭다운 유지. 두 드롭다운 나란히 배치.

```
[유형: 전체 ▼]  [일정: 전체 일정 ▼]
```

| 항목 | 평가 |
|------|------|
| 확장성 | ✅ 버튼 행 제거로 레이아웃 안정 |
| 기능 보존 | ✅ 동일 |
| 구현 복잡도 | 매우 낮음 (버튼→select 교체만) |
| 단점 | 컨트롤 2개 유지, 색상 도트 표현 어려움 |

---

## C안 — 색상 도트 아이콘 버튼 (텍스트 제거)

텍스트 레이블을 제거하고 색상 원(●)만 표시. 활성화 시 테두리 강조.

```
[●전체] [●] [●] [●]   ← 도트만, 툴팁으로 유형명 표시
```

| 항목 | 평가 |
|------|------|
| 확장성 | △ 유형 6~7개 이상이면 여전히 줄바꿈 |
| 기능 보존 | ✅ |
| 구현 복잡도 | 낮음 |
| 단점 | 처음 사용자는 어떤 유형인지 불명확, 툴팁은 모바일 미지원 |

---

## D안 — 가로 스크롤 Chip 바

버튼 행을 `overflow-x: auto; flex-wrap: nowrap`으로 변경. 많아져도 한 줄 유지.

```
← [전체] [일상] [여행] [경조] [기념일] [명절] … →
```

| 항목 | 평가 |
|------|------|
| 확장성 | △ 스크롤로 해결하지만 숨겨진 항목 인지 어려움 |
| 기능 보존 | ✅ |
| 구현 복잡도 | 매우 낮음 (CSS 1~2줄 변경) |
| 단점 | 스크롤 존재 자체를 모를 수 있음, 근본 해결책 아님 |

## 결론

**A안** 채택 시 UI가 가장 간결해지고 유형 증가에 완전히 대응 가능.  
빠른 개선이 목적이라면 **B안**도 합리적 선택 (코드 변경 최소).
